# 差勤查詢系統開發需求文件

## 專案背景

本系統已有以下 4 張資料表（由前一階段建立）：

| 資料表 | 說明 |
|--------|------|
| `unit_map` | 單位對照表，欄位：`district_code`、`unit_code`、`person_id` |
| `absence` | 缺勤表，欄位：`unit_code`、`person_id`、`absence_date` |
| `personal_leave` | 事假表，欄位：`unit_code`、`person_id`、`leave_date` |
| `sick_leave` | 病假表，欄位：`unit_code`、`person_id`、`leave_date` |

**代號規則說明：**
- 區部代號：2碼英文 + `00000`，例如 `AA00000`
- 單位代號：2碼英文 + `10000`，例如 `HC10000`
- 人員代號：單位前3碼 + 兩碼數字 + 兩碼數字，例如 `HC10312`

---

## 開發需求

請以純 **HTML + CSS + JavaScript** 架構開發前台查詢系統（單一 HTML 檔案，不依賴後端框架）。
資料來源請直接將 SQL 測試資料以 **JavaScript 物件陣列**內嵌於 HTML 中，模擬資料庫查詢行為。

---

## 功能一：單位查詢

### 操作流程
1. 頁面上方提供輸入框，使用者輸入 **單位代號**（如 `HC10000`）後按下查詢按鈕
2. 頁面下方同時顯示以下 **2 張表格**

---

### 1-1 請假表 A（Leave Table A）

**顯示邏輯：**
列出該單位所有人員的請假統計，每人一列。

| 欄位 | 說明 |
|------|------|
| 第一欄：人員代號 | 顯示 `person_id` |
| 第二欄：病假天數 | 計算該人員在 `sick_leave` 表中的記錄筆數（天數） |
| 第三欄：事假天數 | 計算該人員在 `personal_leave` 表中的記錄筆數（天數） |
| 第四欄：溝通期 | 下拉選單，選項為：`（未填）`、`第一溝通期`、`第二溝通期`、`第三溝通期`，預設為`（未填）`，使用者可選擇並儲存至 `localStorage` |

---

### 1-2 缺勤表 A（Absence Table A）

**顯示邏輯：**
列出該單位所有人員的缺勤統計，每人一列。

| 欄位 | 說明 |
|------|------|
| 第一欄：人員代號 | 顯示 `person_id` |
| 第二欄：缺勤天數 | 計算該人員在 `absence` 表中的記錄筆數（天數） |
| 第三欄：處理狀態 | 下拉選單，選項為：`（未填）`、`請假`、`終僱`、`留停`，預設為`（未填）`，使用者可選擇並儲存至 `localStorage` |

---

## 功能二：區部查詢

### 操作流程
1. 頁面上方提供輸入框，使用者輸入 **區部代號**（如 `AA00000`）後按下查詢按鈕
2. 系統從 `unit_map` 查出該區部轄下所有單位（`district_code` 相符的 `unit_code`）
3. 頁面下方同時顯示以下 **2 張表格**，**僅顯示該區部轄下單位，嚴禁顯示其他區部資料**

---

### 2-1 請假表 B（Leave Table B）

**顯示邏輯：**
以單位為單位彙總請假數據，每個單位一列。

| 欄位 | 說明 |
|------|------|
| 第一欄：單位代號 | 顯示 `unit_code`，**可點擊**，點擊後以 Modal 彈窗顯示該單位的**請假表 A** |
| 第二欄：病假超過10天人數 | 該單位中，病假天數 **≥ 10 天** 的人員數量 |
| 第三欄：事假超過20天人數 | 該單位中，事假天數 **≥ 20 天** 的人員數量 |

---

### 2-2 缺勤表 B（Absence Table B）

**顯示邏輯：**
以單位為單位彙總缺勤數據，每個單位一列。

| 欄位 | 說明 |
|------|------|
| 第一欄：單位代號 | 顯示 `unit_code`，**可點擊**，點擊後以 Modal 彈窗顯示該單位的**缺勤表 A** |
| 第二欄：缺勤達7天以上人數 | 該單位中，缺勤天數 **≥ 7 天** 的人員數量 |
| 第三欄：回報率 | 計算公式：`已填寫處理狀態人數 ÷ 缺勤≥7天總人數`，顯示為百分比，例如 `3/7（42.9%）`。**已填寫**指缺勤表A中該人員的處理狀態下拉選單不為`（未填）` |

## 介面設計需求

### 整體佈局
- 頁面頂部：系統標題「差勤管理查詢系統」
- 頁面分為左右兩大區塊或上下兩個查詢區塊：**單位查詢區**、**區部查詢區**
- 各查詢區有各自的輸入框與查詢按鈕
- 查詢結果顯示於各自查詢區下方

### 表格樣式
- 表格需有邊框、交替行背景色（斑馬紋）
- 欄位標題固定，資料列可滾動（若資料筆數多）
- 可點擊的單位代號需有明顯的連結樣式（藍色底線或按鈕樣式）

### Modal 彈窗（點擊單位代號時）
- 彈窗標題顯示「單位 XXXXXXX 詳細資料」
- 內含該單位的請假表A 或 缺勤表A（依來源決定）
- 右上角有關閉按鈕（×）
- 點擊遮罩層也可關閉

### 下拉選單儲存機制
- 所有下拉選單的選擇結果需儲存至 `localStorage`
- 頁面重新整理後應能恢復上次選擇的狀態
- 儲存的 key 格式建議：`leave_status_{person_id}` / `absence_status_{person_id}`

### 回報率顏色提示
- 回報率 < 50%：紅色字體
- 回報率 50%~79%：橙色字體
- 回報率 ≥ 80%：綠色字體
- 分母為 0（無缺勤≥7天人員）時顯示 `N/A`

## 技術規格

- **語言：** 純 HTML5 + CSS3 + Vanilla JavaScript（ES6+）
- **不使用任何外部框架或 CDN**（無 jQuery、Bootstrap、Vue、React 等）
- **所有資料內嵌**於 JavaScript 變數中，模擬資料庫
- **單一 HTML 檔案**交付，命名為 `attendance.html`
- 資料結構範例（請依實際產生的測試資料填入）：

```javascript
const unitMap = [
  { district_code: "AA00000", unit_code: "HC10000", person_id: "HC10312" },
  // ... 其餘資料
];

const absenceData = [
  { unit_code: "HC10000", person_id: "HC10312", absence_date: "2025-03-15" },
  // ... 其餘資料
];

const personalLeaveData = [
  { unit_code: "HC10000", person_id: "HC10312", leave_date: "2025-02-10" },
  // ... 其餘資料
];

const sickLeaveData = [
  { unit_code: "HC10000", person_id: "HC10312", leave_date: "2025-01-05" },
  // ... 其餘資料
];
```
