# 勞動日報 Prompt

## 任務
每日整理台灣勞動時事，最多 5 則，寫成結構化筆記推上網站。

---

## 一、找新聞的方法

**來源不限定**，使用者會自行提供當日素材，可能來自：
- 新聞網站（自由、ETtoday、民視、東森、TVBS 等任何媒體）
- 勞動部 / 縣市勞動局官方公告
- 社群平台（Threads、Facebook、Line 群組截圖等）
- 律師或勞工團體的評論貼文

**收到素材後的處理原則**：直接從使用者提供的內容中篩選，不需要另行搜尋。若素材不足 1 則，才視情況主動補充。

**優先收錄的事件類型**
1. 政府裁罰（勞動部 / 縣市勞動局開罰紀錄）
2. 官方政策 / 法規修正公告
3. 重大爭議事件（霸凌、欠薪、違法解僱）
4. 勞檢結果 / 違法行業警示

**不收錄**
- 純報導無違法事實的勞資新聞
- 重複昨日已收錄的事件
- 超過 3 天的舊聞（除非今日有新進展）

---

## 二、篩選原則（最多 5 則）

優先順序：
1. 有明確裁罰金額或行政處分者
2. 違法法條清楚、有學習價值者
3. 影響範圍廣（大型雇主、政府機關、連鎖企業）
4. 近期爭議熱度高、社群討論多者

5 則額滿時，捨棄地域偏遠、金額小、無法條依據的事件。

---

## 三、寫作格式

**frontmatter**
```yaml
---
title: '勞動日報 YYYY-MM-DD｜[3–4 個關鍵字摘要]'
description: '本日 N 則：一句話列出各則主題。'
category: '勞動'
subcategory: '勞動日報'
pubDate: 'YYYY-MM-DD'
---
```

**每則結構（嚴格控制在 5–8 行內）**
```markdown
## N. [單位或事件名稱]，[一句話說明結果]

[2–3 句說明事件經過，去除冗詞。]

**違反法條**：《法規名稱》第 XX 條（條文主旨）

> 一句話學習點，說清楚勞工或雇主應注意什麼。
```

**文末來源**
```markdown
---
**來源**：媒體A、媒體B、Threads @xxx
```

**寫作原則**
- 每則不超過 8 行
- 法條直接標明，不用「相關法規」帶過
- 學習點用 blockquote（`>`），一句話，站在勞工或雇主視角擇一
- 事件名稱直接說結果，不用「疑似」、「可能」等模糊語

---

## 四、推上網站

**檔案路徑**
```

```

**指令**
```bash
git add src/content/notes/勞動/勞動日報/勞動日報-YYYY-MM-DD.md
git commit -m "add: 勞動日報 YYYY-MM-DD（X則）"

```
